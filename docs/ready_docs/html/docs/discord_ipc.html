Discord IPC wrapper¶

Simple Discord IPC wrapper that gives opportunity to use Discord Rich Presence for example.

class DiscordIPC¶

Connecting to Discord app installed locally, sending commands, handling responses.

__init__(self, client_id)¶

DiscordIPC constructor.

Parameters:client_id (string) – unique Discord client ID generated for application created in Discord web panel for developers
get_current_time(self)¶

Get current time from system clock.

Returns:current time from system clock in seconds
Return type:float
get_system_name(self)¶

Getting system name and checking if running OS is supported. Supported OS: Windows, Linux, MacOS.

Returns:running OS name (lowercase)
Return type:string
get_ipc_socket(self)¶

Searching for Discord IPC socket. Different localizations to search for on different platforms.

Returns:path to Discord IPC socket
Return type:string
generate_uuid(self)¶

Generating unique uuid.

Returns:unique uuid
Return type:uuid
connect(self)¶

Trying connect to Discord. Connection is established when Discord recives initial message and responses for the message.

disconnect(self)¶

Disconnecting from Discord by terminating asynchronous methods and Thread that keeps connection alive, reseting all components.

async handshake(self)

Handshaking with Discord (negotiation between two communicating participants) in asynchronous way.

keep_conncetion_alive(self)¶

Keeping connection alive. No timeout functions for fast terminating.

async proper_close(self)

Using asyncio.ProactorEventLoop() on Windows causes some errors while terminationg pipe connection and after that immediately closing main event loop. This easy hack repairs it.

async read_data(self)

Reciving and decoding data from Discord in asynchronous way.

send_data(self, opcode, payload)¶

Encoding data to send and sending encoded data to Discord app via Discord IPC socket.

Parameters:
  • opcode – Discord opcode that defines payload type
  • payload (string) – data that will be send to Discord app (Discord commands) in proper appearance described on Discord website - developers section
send_simple_rich_presence(self, activity_details, activity_state)¶

Creating and sending simple Discord Rich Presence payload to Discord.

Parameters:
  • activity_details (string) – main description of activity
  • activity_state (string) – additional description of activity
send_complex_rich_presence(self, large_text, large_image, small_text, small_image, activity_details, activity_state)¶

Creating and sending complex (full) Discord Rich Presence payload to Discord.

Parameters:
  • large_text (string) – text to display when large image is hovered
  • large_image (string) – name of large image asset seted on Discord developers website
  • small_text (string) – text to display when small image is hovered
  • small_image (string) – name of small image asset seted on Discord developers website
  • activity_details (string) – main description of activity
  • activity_state (string) – additional description of activity
